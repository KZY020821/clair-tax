# Python Quality Skill

## Purpose
Use this skill for maintainable Python implementation.

## Core rules
- Prefer readable functions and explicit names.
- Keep modules focused.
- Add type hints where they improve clarity.
- Avoid mixing I/O, validation, and business logic unnecessarily.

## Output expectations
- Write Python that is clear, testable, and easy to evolve.
