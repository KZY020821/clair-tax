# Frontend UX Skill

## Purpose
Use this skill for polished UI work.

## Core rules
- Prioritize clarity, hierarchy, and responsiveness.
- Use semantic HTML when possible.
- Ensure keyboard-accessible flows.
- Do not rely on color alone to communicate meaning.

## UI expectations
- Always consider loading, empty, error, disabled, and success states.
- Keep copy concise and explicit.
- Use field labels and helper text that reduce confusion.

## Tables and forms
- Make dense information readable.
- Keep forms grouped logically.
- Preserve user input on validation errors when practical.

## Output expectations
- Prefer polished UI over bare scaffolding.
- Keep interfaces calm and easy to scan.
